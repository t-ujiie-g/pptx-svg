# pptx-svg — PPTX 完全互換レンダリングライブラリ

**目標**: OOXML PresentationML (ECMA-376 / ISO 29500) 準拠の PPTX → SVG レンダリング + PPTX エクスポート
**スコープ外**: アニメーション (`p:timing`)、トランジション (`p:transition`)、マクロ/VBA

**互換性方針**: 取り込んだ要素はすべてデータモデルに保持し、エクスポート時に再出力する。
レンダリング不可の要素も round-trip で欠落させない。表示は可能な範囲で 2D 近似する。

## 2. データ保持 — レンダリング限定、round-trip 必須 [P2]

以下は完全なレンダリングは困難だが、データモデルに取り込んでエクスポート時に保持する。
可能な範囲でフォールバック表示 (プレースホルダ画像、簡易表示) を行う。

### フォールバック画像で表示可能

- [ ] **SmartArt** (`dgm:*` DiagramML) — フォールバック画像 (`mc:AlternateContent` 内) で表示 + 原 XML 保持
- [ ] **OLE / 埋め込み** (`p:oleObj`) — フォールバック画像で表示 + 原 XML 保持
- [ ] **メディア** (`p:oleObj` ビデオ/オーディオ) — ポスターフレーム画像で表示 + 関連 rels 保持

### バイナリ保持 (表示不可)

- [ ] **EMF / WMF** — ZIP 内バイナリ保持 + rels 保持 (ブラウザでデコード不可)
- [ ] **TIFF** — ZIP 内バイナリ保持 + rels 保持 (ブラウザ `<img>` 非対応)

### パース + データ保持 (簡略表示)

- [ ] **数式** (OMML `m:oMath`) — 原 XML 保持 + プレーンテキスト抽出でフォールバック表示
- [ ] **埋め込みフォント** (`a:fontScheme` + fontdata) — フォント定義保持 + システムフォントフォールバック

## スコープ外

以下は対応しない。アニメーションとトランジションはround-trip 保持だけ対応したい。

- アニメーション (`p:timing`) — 静的レンダリングの範囲外
- トランジション (`p:transition`) — 静的レンダリングの範囲外
- マクロ / VBA — セキュリティ上の理由で非対応
